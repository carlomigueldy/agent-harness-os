THE CLOSED SET · 5 SUBSYSTEMS

Five subsystems. One kernel.

The harness is a closed set of five. The numbering is the set, not decoration.

  • 0x01 · INSTRUCTIONS Rules, constraints, definition of done.

    AGENTS.md · CLAUDE.md · .agents/context/

  • 0x02 · TOOLS Slash commands, skills, scripts, MCP tools.

    .claude/commands/ · .claude/skills/

  • 0x03 · ENVIRONMENT Runtimes, env vars, local setup.

    init.sh · .agents/context/environment.md

  • 0x04 · STATE Progress, decisions, handovers, a machine-readable feature list.

    claude-progress.md · feature_list.json

  • 0x05 · FEEDBACK Review loops, verification, scoring rubric, retrospectives.

    evaluator-rubric.md · .agents/logs/verification.md

DISPATCH

Match the coordination to the work.

Escalate only as far as the task earns. Most work is one agent. Reach for more structure when — and only when — it pays for its coordination cost.

single agent → subagents → agent teams → dynamic workflows
planner · architect · reviewer=opus
implementer · tester=sonnet
scout=haiku

Each session runs in its own git worktree — isolated, never stepping on main.

AUTONOMY, WITH A HARD STOP

Every loop has a ceiling.

/build-loop max 06
/review-loop max 10
/fix-loop max 10
/test-loop max 06

Every loop has a hard cap and a stop condition. Hit the ceiling and it documents what’s left and hands off — it never spins forever.

THE STRICT GATE

Nothing passes at 9.

A reviewer scores every change 1–10 and returns one verdict. In an autonomous loop, PASS means 10/10 with zero Critical or Major issues. Anything less revises or blocks.

10 / 10
→ PASS
verdicts PASS REVISE BLOCK

strict mode: 10/10 with zero Critical or Major — or it doesn’t ship
